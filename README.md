## مشخصات فنی و ساختار پروژه

* **زبان برنامه‌نویسی:** C# (سی‌شارپ) مبتنی بر بستر **.NET 10**
* **نوع خروجی:** Web API خالص با قالب ساختاری **JSON**

---

### 1. معماری میکروسرویس (Microservices Architecture)
پروژه از دو میکروسرویس کاملاً مستقل، مجزا و مجرد تشکیل شده است که هر کدام بر روی پروسس و پورت شبکه اختصاصی خود اجرا می‌شوند:
* **میکروسرویس کاربران (UserService):** مدیریت فرآیند ثبت‌نام، پروفایل و حذف و آپدیت کاربران.
* **میکروسرویس کتاب‌ها (BookService):** مدیریت کاتالوگ کتاب‌ها، وضعیت موجودی، ویرایش و حذف آن‌ها.

### 2. معماری طراحی دامنه-محور (Domain-Driven Design)
هر میکروسرویس برای حفظ استقلال و لایه‌بندی استاندارد، به لایه‌های اصولی DDD تفکیک شده است:
* **لایه Domain:** هسته اصلی میکروسرویس، شامل موجودیت‌ها (Entities) و منطق خالص کسب‌وکار بدون وابستگی به فریم‌ورک‌های خارجی.
* **لایه Application:** مدیریت فرآیندها، جریان داده‌ها و هندلرهای مربوط به الگوهای CQRS.
* **لایه Infrastructure:** مدیریت داده‌ها و منابع اطلاعاتی.
* **لایه Api:** لایه ارتباطی (Controllers) جهت مواجهه با درخواست‌های HTTP و تولید خروجی JSON.

### 3. الگوی معماری CQRS (Command Query Responsibility Segregation)
جهت بهینه‌سازی کارایی و تفکیک وظایف، عملیات‌های خواندن و نوشتن کاملاً از یکدیگر جدا شده‌اند:
* **Commands (عملیات تغییر وضعیت):** کلاس‌های مجزا برای ایجاد، ویرایش و حذف (مانند `CreateBookCommand` ،`UpdateBookCommand` ،`DeleteBookCommand`).
* **Queries (عملیات پرس‌وجو):** کلاس‌های مجزا برای خواندن اطلاعات (مانند `GetBooksQuery` ،`GetUsersQuery`).

### 4. الگوهای طراحی (Design Patterns)
* **Factory Method Pattern:** در لایه Domain برای ساخت کپسوله‌شده و قانون‌مند موجودیت‌های `User` و `Book` بدون دسترسی مستقیم به سازنده کلاس (Constructor) جهت اعمال دقیق ولیدیشن‌ها.
* **Singleton Pattern:** استفاده از الگوی سینگلتون Thread-Safe برای لایه داده (`FakeDbContext` و `FakeBookDbContext`) جهت اطمینان از وجود تنها یک نمونه (Single Instance) از منبع داده در طول حیات اجرای هر میکروسرویس.

### 5. اصول Clean Code ،SOLID و DRY
* **اصل SRP (تک‌وظیفه‌ای):** هر کلاس یا کنترلر فقط و فقط یک وظیفه مشخص دارد.
* **اصل DIP (وارونگی وابستگی):** لایه‌های بالا به جزئیات لایه‌های پایین وابسته نیستند.
* **اصل DRY:** جلوگیری از هرگونه تکرار کد در ساختار هندلرها و مدل‌های کامند.

##  لایه بانک اطلاعاتی (Database Layer)

* **بانک اطلاعاتی هدف:** **SQL Server**
* **مکانیزم پیاده‌سازی:** به دلیل معماری توزیع‌شده میکروسرویس بدون نیاز به درگیری با پیکربندی‌های سنگین بانک اطلاعاتی محلی، اتصال‌های شبکه یا فرآیند کپی کردن فایل‌های دیتابیس، لایه ذخیره‌سازی داده‌های SQL Server با استفاده از پترن‌های Singleton و Repository به صورت **In-Memory (در حافظه RAM)** پیاده‌سازی شده است. این ساختار فرآیند عملیات تکمیلی CRUD را دقیقاً بر اساس رفتارهای استاندارد تراکنشی مدیریت می‌کند.

## پیاده‌سازی کامل عملیات CRUD (خواند، نوشتن، به‌روزرسانی، حذف)

یک فرآیند کامل و جامع اجرایی بر روی هر دو میکروسرویس پیاده‌سازی شده که هر ۴ عملیات حیاتی CRUD را پشتیبانی می‌کند.

### راهنمای تست عملیات با ابزارهای Postman و Swagger (OpenAPI)

برای تست، ابتدا وارد پوشه API هر میکروسرویس شده و دستور `dotnet run` را بزنید. سپس درخواست‌های زیر را در Postman ارسال کنید:

#### ۱. میکروسرویس کاربران ([UserService](UserService.Api/) - پورت 5163):
* **Create (نوشتن - POST):** `http://localhost:5163/api/users`
    ```json
    { "username": "baran", "email": "baran@email.com", "fullName": "باران علوی" }
    ```
* **Read (خواندن - GET):** `http://localhost:5163/api/users` (دریافت لیست کامل کاربران)
* **Update (به‌روزرسانی - PUT):** `http://localhost:5163/api/users/{id}`
    ```json
    { "fullName": "باران علوی (اصلاح شده)" }
    ```
* **Delete (حذف - DELETE):** `http://localhost:5163/api/users/{id}`

#### ۲. میکروسرویس کتاب‌ها (BookService [BookService](BookService.Api/) - پورت 5288):
* **Create (نوشتن - POST):** `http://localhost:5288/api/books`
    ```json
    { "title": "بوستان", "author": "سعدی" }
    ```
* **Read (خواندن - GET):** `http://localhost:5288/api/books` (دریافت لیست کتاب‌ها شامل دیتای پایه‌ای شاهنامه)
* **Update (به‌روزرسانی - PUT):** `http://localhost:5288/api/books/{id}`
    ```json
    { "title": "شاهنامه فردوسی نسخه کامل", "author": "حکیم ابوالقاسم فردوسی" }
    ```
* **Delete (حذف - DELETE):** `http://localhost:5288/api/books/{id}`

*نکته: مستندات رسمی ساختارها طبق استانداردهای دات‌نت ۱۰ در آدرس `/openapi/v1.json` هر سرویس در دسترس است.*
